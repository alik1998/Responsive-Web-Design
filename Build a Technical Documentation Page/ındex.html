Introduction

JavaScript, platformlar arası, nesne yönelimli bir kodlama dilidir. Küçük ve hafif bir dildir. Bir ana bilgisayar ortamında (örneğin bir web tarayıcısı), JavaScript, onlar üzerinde programlı kontrol sağlamak için ortamındaki nesnelere bağlanabilir.

JavaScript, Dizi, Tarih ve Matematik gibi standart bir nesne kitaplığı ile operatörler, kontrol yapıları ve ifadeler gibi temel dil öğeleri kümesini içerir. Temel JavaScript, ek nesnelerle desteklenerek çeşitli amaçlarla genişletilebilir; Örneğin:

  • İstemci tarafı JavaScript, bir tarayıcıyı ve onun Belge Nesne Modelini (DOM) kontrol etmek için nesneler sağlayarak çekirdek dili genişletir. Örneğin, istemci tarafı uzantıları, bir uygulamanın bir HTML formuna öğeler yerleştirmesine ve fare tıklamaları, form girişi ve sayfada gezinme gibi kullanıcı etkinliklerine yanıt vermesine olanak tanır.
  • Sunucu tarafı JavaScript, bir sunucuda JavaScript'in çalıştırılmasıyla ilgili nesneler sağlayarak çekirdek dili genişletir. Örneğin, sunucu tarafı uzantıları, bir uygulamanın bir veritabanıyla iletişim kurmasına, bir uygulamadan diğerine bilgi sürekliliği sağlamasına veya bir sunucuda dosya manipülasyonları gerçekleştirmesine olanak tanır.

JavaScript and Java

JavaScript ve Java bazı yönlerden benzer ancak bazı yönlerden temelde farklıdır. JavaScript dili Java'ya benzer ancak Java'nın statik yazımına ve güçlü tür kontrolüne sahip değildir. JavaScript, çoğu Java ifadesi sözdizimini, adlandırma kurallarını ve temel kontrol akışı yapılarını takip eder; bu nedenle, LiveScript'ten JavaScript'e yeniden adlandırılmıştır.

Java'nın bildirimlerle oluşturulan sınıflardan oluşan derleme zamanı sisteminin aksine, JavaScript sayısal, Boolean ve dize değerlerini temsil eden az sayıda veri türüne dayalı bir çalışma zamanı sistemini destekler. JavaScript, daha yaygın olan sınıf tabanlı nesne modeli yerine prototip tabanlı bir nesne modeline sahiptir. Prototip tabanlı model dinamik miras sağlar; yani miras alınan şey, tek tek nesnelere göre değişebilir. JavaScript ayrıca herhangi bir özel bildirim gereksinimi olmayan işlevleri de destekler. İşlevler, gevşek yazılmış yöntemler olarak yürütülen nesnelerin özellikleri olabilir.

JavaScript, Java'ya kıyasla oldukça serbest biçimli bir dildir. Tüm değişkenleri, sınıfları ve yöntemleri bildirmeniz gerekmez. Yöntemlerin genel mi, özel mi yoksa korumalı mı olduğu konusunda endişelenmenize ve arayüzleri uygulamanıza gerek yoktur. Değişkenler, parametreler ve işlev dönüş türleri açıkça yazılmaz.

Hello world

JavaScript yazmaya başlamak için Scratchpad'i açın ve ilk "Merhaba dünya" JavaScript kodunuzu yazın:

function greetMe(yourName) { alert("Hello " + yourName); }
greetMe("World");

Peddeki kodu seçin ve tarayıcınızda açılmasını izlemek için Ctrl+R tuşlarına basın!

Variables

Değişkenleri uygulamanızdaki değerler için sembolik adlar olarak kullanırsınız. Tanımlayıcı adı verilen değişkenlerin adları belirli kurallara uygundur.

Bir JavaScript tanımlayıcısı bir harfle, alt çizgiyle (_) veya dolar işaretiyle ($) başlamalıdır; sonraki karakterler de rakam (0-9) olabilir. JavaScript büyük/küçük harfe duyarlı olduğundan, harfler "A" ile "Z" arası karakterleri (büyük harf) ve "a" ile "z" arası karakterleri (küçük harf) içerir.

Tanımlayıcılarda ISO 8859-1 veya å ve ü gibi Unicode harfleri kullanabilirsiniz. Unicode kaçış dizilerini tanımlayıcılarda karakter olarak da kullanabilirsiniz. Yasal adlara örnek olarak Number_hits, temp99 ve _name verilebilir.

Var anahtar kelimesiyle. Örneğin, var x = 42. Bu sözdizimi hem yerel hem de global değişkenleri bildirmek için kullanılabilir.

Bu sözdizimi hem yerel hem de global değişkenleri bildirmek için kullanılabilir. x = 42. Bu her zaman global bir değişken bildirir. Sıkı bir JavaScript uyarısı oluşturur. Bu varyantı kullanmamalısınız.

Let anahtar kelimesiyle. Örneğin, let y = 13. Bu sözdizimi bir blok kapsamı yerel değişkenini bildirmek için kullanılabilir. Aşağıdaki Değişken kapsamına bakın.

Variable scope

Herhangi bir işlevin dışında bir değişken bildirdiğinizde buna global değişken denir, çünkü geçerli belgedeki diğer kodlarda kullanılabilir. Bir işlev içinde bir değişken bildirdiğinizde, buna yerel değişken denir çünkü yalnızca o işlev içinde kullanılabilir.

ECMAScript 2015'ten önceki JavaScript'in blok ifadesi kapsamı yoktur; bunun yerine, bir blok içinde bildirilen bir değişken, bloğun içinde bulunduğu işleve (veya genel kapsama) göre yereldir. Örneğin, aşağıdaki kod 5'i günlüğe kaydedecektir, çünkü x'in kapsamı, bu durumda bir if ifadesi olan blok değil, x'in içinde bildirildiği işlevdir (veya genel bağlamdır).

if (true) { var x = 5; } console.log(x); // 5

This behavior changes, when using the let declaration introduced in ECMAScript 2015.

if (true) { let y = 5; } console.log(y); // ReferenceError: y is not defined
Global variables

Global değişkenler aslında global nesnenin özellikleridir. Web sayfalarında global nesne window'dur, dolayısıyla window.variable sözdizimini kullanarak global değişkenleri ayarlayabilir ve bunlara erişebilirsiniz.

Sonuç olarak, bir pencere veya çerçevede bildirilen global değişkenlere, pencere veya çerçeve adını belirterek başka bir pencere veya çerçeveden erişebilirsiniz. Örneğin, bir belgede telefonNumber adlı bir değişken bildirilmişse, bu değişkene bir iframe'den parent.phoneNumber olarak başvurabilirsiniz.

Constants

const anahtar sözcüğüyle salt okunur, adlı bir sabit oluşturabilirsiniz. Sabit tanımlayıcının sözdizimi değişken tanımlayıcıyla aynıdır: bir harfle, alt çizgiyle veya dolar işaretiyle başlamalıdır ve alfabetik, sayısal veya alt çizgi karakterleri içerebilir.

const PI = 3.14;

Bir sabit, atama yoluyla değeri değiştiremez veya komut dosyası çalışırken yeniden bildirilemez. Bir değere başlatılması gerekir.

Sabitlere ilişkin kapsam kuralları, izin blok kapsam değişkenlerine ilişkin kurallarla aynıdır. Eğer const anahtar sözcüğü atlanırsa tanımlayıcının bir değişkeni temsil ettiği varsayılır.

Aynı kapsamdaki bir işlev veya değişkenle aynı ada sahip bir sabit bildiremezsiniz. Örneğin:

// THIS WILL CAUSE AN ERROR function f() {}; const f = 5; // THIS WILL CAUSE AN ERROR ALSO function f() { const g = 5; var g; //statements }

Ancak nesne öznitelikleri korunmadığından aşağıdaki ifade sorunsuz olarak yürütülür.

const MY_OBJECT = {"key": "value"}; MY_OBJECT.key = "otherValue";

  • İlkel olan altı veri türü:

    • Boolean. true ve false.
    • hükümsüz. Boş bir değeri belirten özel bir anahtar kelime. JavaScript büyük/küçük harfe duyarlı olduğundan null, Null, NULL veya başka herhangi bir değişkenle aynı değildir.
    • Tanımsız. Değeri tanımsız olan üst düzey bir özellik.
    • Number. 42 or 3.14159.
    • String. "Howdy"
    • Sembol (ECMAScript 2015'te yeni). Örnekleri benzersiz ve değişmez olan bir veri türü.
  • ve Object

Bu veri türleri nispeten küçük miktarlarda olsa da uygulamalarınızla faydalı işlevler gerçekleştirmenizi sağlar. Nesneler ve işlevler dilin diğer temel öğeleridir. Nesneleri değerler için adlandırılmış kaplar olarak, işlevleri ise uygulamanızın gerçekleştirebileceği prosedürler olarak düşünebilirsiniz.

if...else statement

Mantıksal bir koşulun doğru olması durumunda bir ifadeyi yürütmek için if ifadesini kullanın. Koşul yanlışsa bir ifadeyi yürütmek için isteğe bağlı else cümleciğini kullanın. Bir if ifadesi aşağıdaki gibi görünür:

if (condition) { statement_1; } else { statement_2; }

koşul, doğru veya yanlış olarak değerlendirilen herhangi bir ifade olabilir. Neyin doğru ve yanlış olarak değerlendirildiğine ilişkin bir açıklama için Boolean'a bakın. Koşul doğru olarak değerlendirilirse ifade_1 yürütülür; aksi halde ifade_2 yürütülür. ifade_1 ve ifade_2, daha fazla iç içe geçmiş if ifadeleri de dahil olmak üzere herhangi bir ifade olabilir.

Birden fazla koşulun sırayla test edilmesini sağlamak için else if kullanarak ifadeleri aşağıdaki gibi birleştirebilirsiniz:

if (condition_1) { statement_1; } else if (condition_2) { statement_2; } else if (condition_n) { statement_n; } else { statement_last; }

Birden fazla koşulun olması durumunda yalnızca doğru olarak değerlendirilen ilk mantıksal koşul yürütülür. Birden fazla ifadeyi yürütmek için bunları bir blok ifadesi ({ ... }) içinde gruplayın. Genel olarak, özellikle if ifadelerini iç içe yerleştirirken her zaman blok ifadelerini kullanmak iyi bir uygulamadır:

if (condition) { statement_1_runs_if_condition_is_true; statement_2_runs_if_condition_is_true; } else { statement_3_runs_if_condition_is_false; statement_4_runs_if_condition_is_false; }

Koşullu ifadelerde basit atamaların kullanılmaması tavsiye edilir çünkü koda bakıldığında atama eşitlikle karıştırılabilir. Örneğin aşağıdaki kodu kullanmayın:

if (x = y) { /* statements here */ }

Bir atamayı koşullu bir ifadede kullanmanız gerekiyorsa, yaygın bir uygulama, atamanın etrafına ek parantez koymaktır. Örneğin:

if ((x = y)) { /* statements here */ }
while statement

Bir while ifadesi, belirtilen bir koşul doğru olarak değerlendirildiği sürece ifadelerini yürütür. Bir süre ifadesi aşağıdaki gibi görünür:

while (condition) statement

Koşul yanlış olursa, döngü içindeki ifadenin yürütülmesi durdurulur ve kontrol, döngüyü takip eden ifadeye geçer.

Koşul testi, döngüdeki ifade yürütülmeden önce gerçekleşir. Koşul doğru dönerse ifade yürütülür ve koşul yeniden test edilir. Koşul false değerini döndürürse yürütme durdurulur ve kontrol while ifadesini takip eden ifadeye aktarılır.

Birden fazla ifadeyi yürütmek için, bu ifadeleri gruplandırmak üzere bir blok ifadesi ({ ... }) kullanın.

Örnek:

Aşağıdaki while döngüsü, n üçten küçük olduğu sürece yinelenir:

var n = 0; var x = 0; while (n < 3) { n++; x += n; }

Her yinelemede döngü n değerini artırır ve bu değeri x'e ekler. Bu nedenle x ve n aşağıdaki değerleri alır:

  • İlk geçişten sonra: n = 1 ve x = 1
  • İkinci geçişten sonra: n = 2 ve x = 3
  • Üçüncü geçişten sonra: n = 3 ve x = 6

Üçüncü geçiş tamamlandıktan sonra n < 3 koşulu artık doğru olmadığı için döngü sona erer.

Function declarations

Bir işlev tanımı (aynı zamanda işlev bildirimi veya işlev ifadesi olarak da adlandırılır), function anahtar sözcüğünden ve ardından gelenlerden oluşur:

  • Fonksiyonun adı.
  • A list of arguments to the function, enclosed in parentheses and separated by commas.
  • The JavaScript statements that define the function, enclosed in curly brackets, { }.

Örneğin, aşağıdaki kod kare adında basit bir işlevi tanımlar:

function square(number) { return number * number; }

Kare fonksiyonu sayı adı verilen bir argüman alır. İşlev, işlevin bağımsız değişkeninin (yani sayının) kendisiyle çarpılarak döndürülmesini söyleyen bir ifadeden oluşur. Return deyimi, işlevin döndürdüğü değeri belirtir

function square(number) { return number * number; }

Kare fonksiyonu sayı adı verilen bir argüman alır. İşlev, işlevin bağımsız değişkeninin (yani sayının) kendisiyle çarpılarak döndürülmesini söyleyen bir ifadeden oluşur. Return deyimi, işlevin döndürdüğü değeri belirtir.

return number * number;

Temel parametreler (sayı gibi) işlevlere değere göre aktarılır; değer fonksiyona iletilir, ancak fonksiyon parametrenin değerini değiştirirse bu değişiklik global olarak veya çağıran fonksiyona yansımaz.
